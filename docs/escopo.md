# Escopo do Projeto

## Requisitos Funcionais
- Permitir ao usuário inserir cidade de origem e destino
- Permitir seleção de ida ou ida e volta
- Permitir adicionar cidades intermediárias (multitrechos)
- Permitir seleção de datas para cada trecho
- Buscar preços em APIs gratuitas de passagens aéreas
- Exibir resultados de forma clara, intuitiva e responsiva
- Permitir ordenação e filtragem dos resultados (por preço, duração, companhia)
- Exibir detalhes do voo: preço, datas, horários, companhias, escalas
- Histórico de buscas recentes

## Requisitos Não Funcionais
- Interface responsiva (desktop/mobile)
- Performance adequada para MVP (resposta em até 5s)
- Utilização de Angular e Angular Material
- Integração com pelo menos uma API gratuita de voos
- Código limpo, modular e documentado
- Testes automatizados para componentes principais
- Sem armazenamento de dados sensíveis

## Fora do Escopo (MVP)
- Compra direta de passagens aéreas pela plataforma
- Cadastro/login de usuários
- Integração com métodos de pagamento
- Notificações automáticas de variação de preço
- Suporte a múltiplos idiomas
- Funcionalidades avançadas de recomendação
- Exportação/compartilhamento de resultados
- Deploy em produção pública

## Histórias de Usuário
- Como usuário, quero buscar passagens entre cidades para planejar minha viagem.
- Como usuário, quero adicionar cidades intermediárias para criar roteiros personalizados.
- Como usuário, quero escolher entre ida ou ida e volta.
- Como usuário, quero ver os melhores preços disponíveis para as datas escolhidas.
- Como usuário, quero visualizar detalhes dos voos encontrados.
- Como usuário, quero filtrar e ordenar os resultados conforme minha preferência.
- Como usuário, quero acessar rapidamente minhas buscas recentes.
